MrsJone 能量衡算工具 v4.9
Aspen Plus 数据自动提取与报告生成工具
基于 Python + Tkinter 开发的Aspen Plus 专用能量衡算工具，一键连接 Aspen Plus 软件，自动提取流程模块的流股数据、热负荷、轴功率等关键参数，批量生成标准化 Word 汇总报告 + Excel 明细文件，自动完成能量平衡验证，大幅提升化工流程数据处理效率。
✨ 核心功能
Aspen 无缝对接：通过 COM 接口直连 Aspen Plus，读取 .bkp 备份文件，自动运行模拟刷新数据
批量模块提取：支持 Ctrl 多选流程模块，一键提取所有选中模块的进出料流股数据
自定义导出字段：可自由勾选流股名称、温度、压力、流量、焓值等 8 项核心参数
双格式报告输出
Word 汇总报告：包含所有模块的负荷表、流股明细、能量平衡验证，直接用于文档编写
Excel 独立文件：每个模块生成专属 Excel，含 3 张工作表（负荷表 / 流股明细 / 平衡验证）
实时数据预览：内置表格实时展示提取的所有流股数据，支持清空查看
智能能量计算：自动计算进料总焓、出料总焓、热负荷、轴功率、能量平衡误差
可视化操作界面：状态指示灯、进度条、日志输出，操作全程可视化
支持主流模块：兼容换热器、塔器、泵、压缩机、闪蒸、反应器等常用模块
📋 运行环境要求
1. 软件环境
操作系统：Windows 全系列（仅支持 Windows，依赖 Aspen COM 接口）
Aspen Plus：已正常安装（V8.4 及以上版本均可）
Python：3.8 ~ 3.11 版本
2. 必须安装的 Python 依赖库
bash
运行
pip install python-docx pandas pywin32
python-docx：生成 Word 报告
pandas：数据处理与 Excel 导出
pywin32：Aspen Plus 接口通信
tkinter：Python 自带 GUI 库，无需额外安装
🚀 快速使用步骤
步骤 1：准备文件
将 Aspen Plus 流程保存为 .bkp 备份文件
确保 Aspen Plus 已正常运行 / 可正常启动
步骤 2：启动工具
运行 Python 脚本，打开 GUI 操作界面
步骤 3：配置路径
选择 BKP 文件：点击按钮选中流程备份文件
选择输出目录：指定报告保存的文件夹
步骤 4：连接 Aspen
勾选「显示 Aspen 界面」（可选）
点击「连接 Aspen (读取模块)」
等待状态指示灯变为绿色，代表连接成功
步骤 5：选择模块与导出选项
在模块列表中按住 Ctrl 多选需要提取的模块
勾选需要生成的报告类型（Word/Excel）
勾选需要提取的流股数据字段
步骤 6：开始提取
点击「开始提取选中模块」，等待进度条完成
提取完成后自动切换到数据预览面板
弹出提示后可直接打开输出文件夹查看报告
步骤 7：断开连接
使用完成后点击「断开 Aspen」，关闭工具即可

📄 导出文件说明
1. Word 汇总报告
文件名：能量衡算报告_时间戳.docx包含内容：
报告标题、导出时间、归属单位
每个模块独立分页：模块名称、类型、负荷数据、流股明细、能量平衡验证
标准化表格格式，直接用于课程设计 / 毕业设计 / 工程报告
2. Excel 独立文件
每个模块生成：模块名-能量衡算.xlsx包含 3 张工作表：
表 1 - 负荷表：轴功率 W、热负荷 Q
表 2 - 流股明细：自定义勾选的所有流股参数
表 3 - 平衡验证：进料焓 Hin、出料焓 Hout、能量平衡误差
⚠️ 常见问题与注意事项
连接 Aspen 失败
检查 Aspen Plus 是否正常安装
关闭 Aspen Plus 重新尝试连接
.bkp 文件路径不要包含中文、空格、特殊字符
依赖库报错
按照「依赖安装」命令重新安装库
提示 缺少 python-docx 时，执行 pip install python-docx
数据提取为 0
连接 Aspen 后工具会自动运行模拟，等待模拟完成
检查 Aspen 流程是否收敛、流股是否正确连接
权限问题
以管理员身份运行 Python 脚本
输出目录选择非系统盘（如 D 盘）
运行卡顿
大型流程首次连接 Aspen 需等待 10~30 秒（正常初始化）
关闭其他占用内存的程序
